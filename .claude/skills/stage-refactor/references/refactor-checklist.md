# 阶段重构检查清单

五维度检查标准，用于分析现状和验证交付。

---

## 维度一：结构格式

| 检查项 | 要求 |
|--------|------|
| YAML frontmatter | name + description 字段完整 |
| 目标章节 | 目标清单 + 核心原则 |
| 前置验证 | 文件检查 + 失败处理（非首阶段必需） |
| 阶段完成标志 | 验证顺序表格，状态驱动 |
| 流程速查 | 子流程引用汇总（有子流程时必需） |
| 任务列表 | 任务→步骤层级 |
| Red Flags | 违规行为表格（3-5条） |
| 常见错误 | 错误/原因/解决方案表格 |
| 变量速查 | 引用 SKILL.md 或独立列出 |

---

## 维度二：交互友好

| 检查项 | 要求 |
|--------|------|
| 确认点标注 | 需用户确认处标注 ⚠️ |
| 目的说明 | 交互操作说明目的 |
| 选项清晰 | 选项表格格式 |
| 状态表完整 | 状态表格含后续动作列 |

---

## 维度三：内容精简

| 检查项 | 要求 |
|--------|------|
| 示例数量 | 合理，不过多 |
| 无重复内容 | 不重复解释相同概念 |
| 表格优先 | 列表/段落可改为表格 |
| 无过度解释 | 解释简洁，不冗长 |

---

## 维度四：逻辑合理

| 检查项 | 要求 |
|--------|------|
| 任务顺序 | 符合业务逻辑 |
| 步骤粒度 | 适中，不过细或过粗 |
| 判断条件 | 完整覆盖分支 |
| 命令参数 | 正确且最新 |
| 业务约束 | 体现必需约束 |

> 此维度涉及业务逻辑，发现问题后需询问用户确认。

---

## 维度五：阶段特有

| 检查项 | 适用场景 |
|--------|----------|
| 前置验证完整 | 非首阶段（eval-build/set/execute） |
| 完成标志顺序正确 | 所有阶段 |
| 流程速查表完整 | 有子流程调用的阶段 |
| 任务→步骤层级清晰 | 所有阶段 |

---

## 字数控制

| 文档类型 | 目标 |
|----------|------|
| 阶段文档（eval-*.md） | < 2000 字 |

超出时优先精简示例和解释段落。
