# 阶段文档书写规范

本文档定义阶段文档（eval-init.md、eval-build.md、eval-set.md、eval-execute.md）的书写规范，确保结构统一、交互友好、内容精简。

---

## 全局要求

阶段文档调整的核心目标：

| 目标 | 说明 |
|------|------|
| **结构清晰** | 章节顺序固定，必备章节不遗漏 |
| **层级分明** | 阶段→任务→步骤三层结构 |
| **交互友好** | 用户知目的、选项有补充、确认标注 ⚠️ |
| **内容简洁** | 字数 < 2000，引用不重复 |
| **状态驱动** | 验证顺序，文件存在决定执行路径 |
| **前置可追溯** | 前置验证明确，失败返回对应阶段 |

---

## 一、宏观结构

### 1.1 必备章节顺序

| 序号 | 章节名称 | 必备性 | 说明 |
|------|----------|--------|------|
| 1 | YAML frontmatter | 必须 | name + description |
| 2 | 标题 + 阶段描述 | 必须 | 一级标题 + 陈述式描述 |
| 3 | 目标 | 必须 | 阶段目标 + 核心原则 |
| 4 | 前置验证 | 可选 | 非首阶段必备 |
| 5 | 何时使用 | 必须 | 简单一句描述 |
| 6 | 阶段完成标志 | 必须 | 验证顺序表格 |
| 7 | 流程速查 | 可选 | 有子流程时必备 |
| 8 | 任务列表 | 必须 | 三层结构（任务→步骤） |
| 9 | Red Flags | 必须 | 表格形式（含常见错误） |
| 10 | 变量速查 | 必须 | 引用 SKILL.md |

### 1.2 文档结构模板

```markdown
---
name: {阶段名称}
description: Use when {触发条件描述}
---

# {阶段名称}

{陈述式阶段描述}

## 目标

{阶段目标}。

核心原则：**{核心原则}**。

---

## 前置验证（可选）

{前置文件检查}。验证失败则返回对应前置阶段。

---

## 何时使用

- {使用场景1}
- {使用场景2}

---
```

## 二、细节要求

### 2.1 YAML Frontmatter

| 字段 | 要求 |
|------|------|
| `name` | 阶段英文名称，与文件名一致（eval-init、eval-build 等） |
| `description` | 触发条件描述，使用 "Use when..." 格式 |

**正确示例**：
```yaml
---
name: eval-build
description: Use when initialization completed and need to configure evaluation dimensions, judges, or process evaluation datasets
---
```

---

### 2.2 目标章节

| 要求 | 说明 |
|------|------|
| 阶段目标 | 使用箭头符号表示阶段走向 |
| 核心原则 | 加粗标注，体现执行策略 |
| 输出说明 | 有输出文件时标注"**输出**" |

**示例**：
```markdown
## 目标

完成评测场景确认、维度配置、评委配置后，进入评测集处理阶段。

核心原则：**按序执行，用户确认优先，配置校验后置**。
```

---

### 2.3 前置验证章节

| 要求 | 说明 |
|------|------|
| 必备性 | 非首阶段（eval-build/set/execute）必备 |
| 格式 | 文件检查 + 失败处理 |
| 失败处理 | 返回对应前置阶段 |

**示例**：
```markdown
## 前置验证

`auth.json` 存在、`eval-dimension.json` 存在、`eval-judge.json` 存在、`{session-id}/` 目录存在。验证失败则返回对应前置阶段。
```

---

### 2.4 阶段完成标志章节

| 要求 | 说明 |
|------|------|
| 格式 | 验证顺序表格 |
| 验证条件 | 文件路径或状态检查 |
| 不满足时执行 | 对应任务编号 |
| 结束语 | "全部通过后，进入下一阶段" |

**示例**：
```markdown
## 阶段完成标志

| 验证条件 | 不满足时执行 |
|----------|--------------|
| `eval-dimension.json` 存在 | 任务1、任务2 |
| `eval-judge.json` 存在 | 任务3 |

全部通过后，进入评测集处理阶段。
```

---

### 2.5 流程速查章节

| 要求 | 说明 |
|------|------|
| 适用场景 | 阶段内有子流程调用时必备 |
| 表格格式 | 编号 + 名称 + 文档位置 + 调用时机 |
| 文档位置 | 使用相对路径链接 |

---

### 2.6 任务列表章节

| 要求 | 说明 |
|------|------|
| 层级结构 | 阶段→任务→步骤（三层） |
| 任务格式 | `### 任务N：{任务名称}` |
| 任务目标 | 每个任务开头标注"**目标**" |
| 任务输出 | 有输出文件时标注"**输出**" |
| 步骤格式 | `#### 步骤N：{步骤名称}` |
| 确认标注 | 不可跳过的确认步骤标注 `⚠️` |

---

### 2.7 Red Flags 章节

| 要求 | 说明 |
|------|------|
| 格式 | 表格形式：违规行为 + 简洁理由 |
| 条数 | 5-8条（含常见错误） |
| 常见错误格式 | `{错误} → {解决方案}` |
| 引用 | 底部引用 SKILL.md 通用违规 |

**示例**：
```markdown
## Red Flags

| 违规行为 | 简洁理由 |
|----------|----------|
| 跳过用户确认 | 字段映射需用户确认后才能标准化 |
| 评测集格式错误 → 检查格式 | 文件格式损坏或不支持 |

> 通用违规行为见 [SKILL.md Red Flags](./SKILL.md#red-flags)
```

---

### 2.8 变量速查章节

| 形式 | 适用场景 |
|------|------|
| 引用 SKILL.md | 阶段文档统一使用此形式 |

**标准写法**：
```markdown
## 变量速查

变量定义见 [SKILL.md 变量速查](./SKILL.md#变量速查)
```

---

## 三、判断依据

### 3.1 前置验证章节判断

| 场景 | 选择 |
|------|------|
| 首阶段（eval-init） | 可省略（无前置阶段） |
| 非首阶段（eval-build/set/execute） | 必备 |

### 3.2 流程速查章节判断

**有子流程时必备**，子流程判断标准：
- 任务步骤中明确调用 processes/*.md 文件
- 步骤中存在"执行流程N"的动作

### 3.3 Red Flags 数量要求

- 最少 3 条，最多 5 条
- 优先选择"不可跳过"和"不得提前询问"类错误

---

## 四、字数目标

| 文档类型 | 目标字数 | 原因 |
|----------|----------|------|
| 阶段文档 | < 2000字 | 阶段触发时加载，保持精简 |
| SKILL.md | < 5000字 | 高频加载，主入口文档 |
