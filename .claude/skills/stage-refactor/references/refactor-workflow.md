# 阶段重构执行流程

按序执行三阶段，不得跳过。

---

## 第一阶段：分析现状

### 步骤

1. 读取目标阶段文档
2. 读取 `references/writing-guide.md` 规范
3. 对照规范，输出差异清单（五维度）
4. ⚠️ 询问："是否继续调整？"

**门控条件**：不得进入第二阶段直到用户确认。

### 差异清单格式

清单式输出，非详细表格：

```
维度1-结构格式：{通过/缺失}项：YAML、目标、前置验证...
维度2-交互友好：{问题}项：缺少确认标注...
维度3-内容精简：{问题}项：示例过多...
维度4-逻辑合理：{问题}项：任务顺序待确认...
维度5-阶段特有：{缺失}项：流程速查表...
```

---

## 第二阶段：执行调整

### 步骤

用户确认后执行调整：

| 类型 | 说明 |
|------|------|
| 结构调整 | 补充缺失章节、调整顺序 |
| 格式修正 | YAML、表格、标注格式 |
| 交互优化 | 添加 ⚠️ 标注、目的说明 |
| 内容精简 | 列表改表格、去冗余 |
| ⚠️ 逻辑调整 | 任务顺序、步骤合并（需单独确认） |

每项完成后输出：`✓ {调整项}`

### 门控条件

- 逻辑调整（任务顺序、步骤合并）必须单独确认
- ⚠️ 全部完成后询问："是否进入验证？"
- 不得进入第三阶段直到用户确认

---

## 第三阶段：验证交付

### 步骤

1. 读取调整后文档
2. 加载 `references/refactor-checklist.md` 执行五维度检查
3. 未通过项 → 返回第二阶段
4. 全部通过 → 输出调整摘要
5. ⚠️ 询问："是否保存？"

### 门控条件

不得结束直到用户确认保存。

### 调整摘要格式

```
✓ 结构调整：补充 X 章节
✓ 格式修正：YAML、表格格式
✓ 交互优化：添加 N 处确认标注
✓ 内容精简：移除冗余段落
字数：{调整前} → {调整后}
```
