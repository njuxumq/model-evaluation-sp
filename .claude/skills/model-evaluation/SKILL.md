# 模型评测技能

协助用户完成 AI 模型评测任务的全流程管理。

**核心原则**：自动识别优先，渐进式确认，可恢复执行。

**适用场景**：评测大模型（LLM）输出质量

**不适用场景**：代码质量审查、系统性能测试、模型训练/微调

---

## 处理阶段

| 阶段 | 文档 | 核心任务 |
|------|------|----------|
| 初始化 | [eval-init.md](./eval-init.md) | 环境检测、鉴权验证、会话目录 |
| 构建配置 | [eval-build.md](./eval-build.md) | 场景确认、评测标准、评委配置 |
| 评测集处理 | [eval-set.md](./eval-set.md) | 评测集解析、标准化、上传 |
| 执行评测 | [eval-execute.md](./eval-execute.md) | 任务提交、状态监控、结果展示 |

**执行流程**：阶段1 → 阶段2 → 阶段3 → 阶段4

---

## 交互规范

| 场景 | 处理方式 |
|------|----------|
| 可自动推断 | 直接处理，仅做结果确认 |
| 需用户确认 | 提供候选项供选择（AskUserQuestion） |
| 需用户输入 | 直接请求输入 |

**禁止复合命令**：逐条执行 bash 命令，便于调试和验证。

**必须确认**：维度权重、字段映射、评测点预览、场景识别结果。

> 详细规范见 [交互规范详解.md](./references/交互规范详解.md)

## Red Flags

| 违规行为 | 后果 |
|----------|------|
| 当前阶段未完成时询问后续阶段信息 | 返回当前阶段继续执行 |
| 跳过用户对维度权重、字段映射的确认 | 返回确认步骤重新执行 |
| 执行复合 bash 命令 | 拆分为独立命令重新执行 |

> 详细规范见 [交互规范详解.md](./references/交互规范详解.md)
