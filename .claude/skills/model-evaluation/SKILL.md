# 模型评测技能

协助用户完成 AI 模型评测任务的全流程管理。

**核心原则**：自动识别优先，渐进式确认，可恢复执行。

**适用场景**：评测大模型（LLM）输出质量

**不适用场景**（引导用户使用其他方式）：
- 代码质量审查 → 建议使用代码审查工具
- 系统性能测试 → 建议使用性能测试框架
- 模型训练/微调 → 不属于评测范畴

---

## 处理阶段

| 阶段 | 文档 | 核心任务 |
|------|------|----------|
| 初始化 | [eval-init.md](./eval-init.md) | 环境检测、鉴权验证、会话目录 |

**执行流程**：阶段1 → 阶段2 → 阶段3 → 阶段4 → 评测结束

## 进度展示规范

### 核心原则

| 原则 | 说明 |
|------|------|
| **渐进式披露** | 默认输出简化信息，完整信息按需展示 |
| **情境感知** | 根据用户行为动态调整输出详细程度 |
| **路径可回溯** | 展示"已走过的路径"帮助用户理解上下文 |

### 状态符号

| 符号 | 状态 |
|------|------|
| `✓` | 已完成 |
| `●` | 当前 |
| `○` | 待执行 |
| `~` | 已跳过 |

### 展示层级

| 时机 | 展示内容 |
|------|----------|
| 进入 SKILL | 全流程概览（阶段列表 + 每阶段任务列表） |
| 进入阶段 | 阶段路径 + 当前阶段任务列表 |
| 进入任务 | 任务路径 + 当前任务步骤列表 |
| 进入步骤 | 步骤路径 + 分支判断/步骤描述 |
| 进入流程 | 流程路径 + 流程内步骤列表 |

```bash
mkdir -p dir
cd dir
python script.py
```

## Red Flags

| 常见借口 | 现实纠正 |
|----------|----------|
| "用户应该知道选哪个" | 用户可能不清楚概念，需展示选项 |
| "这个确认可以跳过" | 所有确认步骤都是必需的 |
| "复合命令更快" | 复合命令难以调试，必须拆分 |
| "默认配置就行" | 维度权重影响评测结果，必须确认 |
| "这个信息可以先问" | 当前阶段未完成时不可询问后续阶段信息 |
