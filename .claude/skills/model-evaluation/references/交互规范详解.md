# 交互规范详解

本文档详细定义模型评测技能的交互规范。

---

## 渐进式交互

| 场景 | 处理方式 |
|------|----------|
| 可自动推断 | 直接处理，仅做结果确认 |
| 需用户确认 | 提供候选项供选择（AskUserQuestion） |
| 需用户输入 | 直接请求输入（文本输入） |

### 判断标准

**能否提前列出所有有效答案？**
- 能 → AskUserQuestion（提供选项）
- 不能 → 文本输入（直接请求）

### 帮助触发

| 用户表现 | 处理方式 |
|----------|----------|
| 说"不懂"、"怎么..." | 暂停并解释相关概念 |
| 关键步骤前 | 可附带"了解更多"选项 |
| 多次犹豫 | 主动提供详细说明 |

---

## 命令执行规范

### 禁止复合命令

**错误示例**：
```bash
mkdir -p dir && cd dir && python script.py
```

**正确示例**（逐条执行）：
```bash
mkdir -p dir
cd dir
python script.py
```

### 原因

| 问题 | 说明 |
|------|------|
| 难以调试 | 复合命令失败时，无法定位具体哪一步出错 |
| 难以验证 | 无法在每一步检查状态 |
| 难以回滚 | 无法在失败时停止执行 |

## 可恢复性

### 失败处理流程

```
失败发生
    │
    ├─ 1. 清晰解释原因
    │      └─ "XXX 失败，原因是 YYY"
    │
    ├─ 2. 说明当前进度
    │      └─ "已完成：A、B；未完成：C"
    │
    ├─ 3. 指明下一步操作
    │      └─ "请执行 XXX 后重试"
    │
    └─ 4. 保留已完成产物
           └─ 不删除已生成的文件
```

### 中断恢复

| 中断场景 | 恢复方式 |
|----------|----------|
| 用户主动中断 | 记录当前状态，下次继续 |
| 错误中断 | 提供修复建议，保留进度 |
| 超时中断 | 提供重试选项 |

## 阶段边界

### 禁止跨阶段询问

**错误**：在初始化阶段询问"您的评测场景是什么？"

**正确**：等待初始化完成，进入构建阶段后再询问。

### 例外情况

| 条件 | 允许行为 |
|------|----------|
| 用户明确指明跳转 | 可直接进入指定阶段 |
| 用户主动提供信息 | 可记录，但不主动追问后续阶段信息 |

---

## 确认机制

### 必须确认的场景

| 场景 | 原因 |
|------|------|
| 维度权重配置 | 影响评测结果 |
| 字段映射 | 影响数据解析 |
| 评测点预览 | 影响评测标准 |
| 场景识别结果 | 影响后续配置 |

### 确认方式

| 类型 | 方式 |
|------|------|
| 选择确认 | AskUserQuestion（Y/n/调整） |
| 内容确认 | 展示内容 + 等待回复 |
| 批量确认 | 展示摘要 + 逐项确认选项 |

---

## 错误处理

### 错误信息格式

```
❌ {操作} 失败

原因：{具体原因}

建议操作：
1. {建议1}
2. {建议2}

当前进度：{已完成}/{总步骤}
```

### 常见错误处理

| 错误类型 | 处理方式 |
|----------|----------|
| 网络错误 | 提示重试，提供离线替代方案 |
| 认证失效 | 引导重新授权 |
| 文件格式错误 | 提示格式要求，提供示例 |
| 参数错误 | 说明正确格式，提供修正建议 |
