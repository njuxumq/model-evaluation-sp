# eval-set.md 步骤1新增"生成问题集"选项设计

**日期**：2026-04-02
**状态**：已批准

---

## 背景

当前 `eval-set.md` 步骤1"识别评测集来源"只支持两个分支：
- 已有评测集 → 继续步骤2
- 无法判断 → 询问用户提供评测集

用户需求：新增"用户没有评测集"分支，支持 AI 根据评测场景和维度生成问题集。

---

## 需求澄清

| 问题 | 用户回答 |
|------|----------|
| 触发时机 | 支持用户明确意图，用户可以在任意阶段表示要生成问题集 |
| 流程衔接 | 生成问题集后完整执行解析流程（evalset-parse） |
| category 字段 | 不需要特殊处理，作为可选字段传递 |
| 方案选择 | 方案A：修改步骤1表格 |

---

## 设计决策

### D-1：前置验证位置

前置验证（检查 `eval-dimension.json` 存在）在 `evalset-create.md` 流程内部执行，而非 `eval-set.md` 调用前预先检查。

**理由**：
- 流程内验证更符合"流程自治"原则
- 避免 eval-set.md 与 evalset-create.md 的耦合

## 文档修改清单

### 修改1：eval-set.md 步骤1识别结果表格

**位置**：`.claude/skills/model-evaluation/eval-set.md` 步骤1

**当前内容**：
```
| 识别结果 | 判断依据 | 后续动作 |
|----------|----------|----------|
| 已有评测集 | 文件路径或文件描述 | → 步骤2 |
| 无法判断 | 无相关信息 | → 询问用户 |
```

**修改为**：
```
| 识别结果 | 判断依据 | 后续动作 |
|----------|----------|----------|
| 已有评测集 | 文件路径或文件描述 | → 步骤2 |
| 无评测集 | 用户明确表示没有评测集或需要生成 | → 执行 evalset-create 流程 → 步骤2 |
| 无法判断 | 无相关信息 | → 询问用户 |
```


## 流程衔接说明

```
eval-set.md 步骤1
  → 识别结果：无评测集
    → 执行 evalset-create.md 流程
      → 步骤1：前置验证（检查 eval-dimension.json）
        → 通过 → 步骤2-3 → 输出 evalset-prepared.jsonl
        → 不通过 → 提示用户 → 流程终止
    → 生成完成后 → eval-set.md 步骤2（执行解析流程）
```
