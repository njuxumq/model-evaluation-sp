# 进度展示规范拆分设计

## 背景

当前进度展示规范位于 SKILL.md，占用约 100 行（~1500-2000字）。SKILL.md 目标字数 < 5000 字（高频加载），需要优化 Token 效率。

## 问题分析

| 现状 | 影响 |
|------|------|
| 进度展示规范全在 SKILL.md | 每次加载 SKILL.md 都占用 Token |
| 进入 SKILL 时需展示全流程概览 | 必须立即可用，无法延迟加载 |
| 阶段/任务级进度模板 | 只在进入阶段时需要，可延迟加载 |

## 设计方案：拆分为两层

### 核心层（保留在 SKILL.md）

**内容**：
- 流程感知核心原则（3条）
- 全流程概览模板（进入 SKILL 时展示）
- 状态符号定义（4个）

**预估**：约 25 行（原 100 行减少 75%）

### 详细层（新建 references/进度展示详细规范.md）

**内容**：
- 进入阶段时模板
- 进入任务时模板
- 进入步骤时模板
- 进入流程时模板
- 分支判断格式

**预估**：约 60 行

### 阶段文档引用

各阶段文档在需要展示进度时引用详细规范：

```markdown
> 进度展示模板见 [进度展示详细规范](./references/进度展示详细规范.md)
```

## Token 效率分析

| 文档 | 修改前 | 修改后 | 减少 |
|------|--------|--------|------|
| SKILL.md | ~100 行 | ~25 行 | 75% |
| references/进度展示详细规范.md | 0 | ~60 行 | 新增 |

**效果**：
- SKILL.md 高频加载，Token 消耗显著降低
- 详细规范按需加载，不影响核心流程
- 全流程概览展示能力保持不变

## 实施步骤

1. 修改 SKILL.md：删除详细模板，保留核心层
2. 创建 references/进度展示详细规范.md
3. 在各阶段文档添加引用（按需）
