# 模型评测 Skill 用户交互友好性改进设计

> **For agentic workers:** 本设计文档定义了路径追踪进度条、流程有向图、流转预告机制三项改进。实现时需严格遵循格式规范。

**Goal:** 解决用户在使用 Skill 时的迷失感（不知道"我在哪"、"下一步做什么"）、选择困惑（不知道"该选哪个"、"选了会怎样"）、理解缺失（不知道"为什么要这样做"）三大痛点。

**优先级:** D（步骤嵌套迷失）> A（分支选择迷失）> B（状态判断迷失）> C（理解缺失）

---

## 问题分析

### 用户痛点识别

| 痛点 | 具体场景 | 用户困惑 |
|------|----------|----------|
| **D. 步骤嵌套迷失** | 任务下有步骤，步骤下有判断，判断后又有步骤 | "我刚才做了字段映射确认，现在是在哪个任务的哪个步骤？" |
| **A. 分支选择迷失** | 评测集处理步骤1询问"是否有评测集" | "选了'生成问题集'之后会发生什么？和'提供评测集'有什么不同？" |
| **B. 状态判断迷失** | 解析后检测答案字段状态，系统自动调用不同子流程 | "我的评测集部分有答案，为什么要问我确认类型？" |
| **C. 理解缺失** | 评测方式选择、字段映射确认等步骤 | "为什么要确认字段映射？不确认会怎样？" |

### 当前状态

| 维度 | 状态 | 说明 |
|------|------|------|
| 整体流程感知 | ✅ 已解决 | 流程入口展示四阶段概览 |
| 当前阶段感知 | ✅ 已解决 | 进入阶段时展示信息 |
| 当前任务感知 | ⚠️ 部分解决 | TodoWrite 显示任务，但步骤层级不清晰 |
| 当前步骤感知 | ❌ 待改进 | 步骤嵌套在任务内，用户看不到步骤进度 |
| 选择后流转感知 | ❌ 待改进 | 用户选择后，下一步是什么不明确 |
| 原因解释 | ❌ 待改进 | Why 说明缺失 |

## 设计方案

### 方案概述

采用**层级进度条 + 流程有向图 + 流转预告机制**的组合方案：

| 方案 | 解决痛点 | 核心机制 |
|------|----------|----------|
| 方案1: 路径追踪 | D（步骤嵌套迷失） | 极简定位，仅输出当前位置 |
| 方案2: 流程有向图 | A、B（分支/状态迷失） | 在进入任务/步骤时展示完整分支路径 |
| 方案2: 流转预告 | A、B（分支/状态迷失） | 选择前预告、判断后回顾 |
| 方案3: Why 说明 | C（理解缺失） | 后续迭代添加 |

## 方案1: 路径追踪进度条设计

### 核心目的

**简化进度条，突出有向图**。进度条仅展示当前位置和已走过的路径，完整流程由有向图呈现。

### 格式模板

**进入阶段时**：
```
📍 当前位置：初始化阶段
```

**进入任务时**：
```
📍 当前位置：初始化 > 任务1：环境及依赖检测
```

**进入步骤时**：
```
📍 当前位置：初始化 > 任务1 > 步骤3：执行环境检测
```

**完成步骤时**（可选，仅在关键节点输出）：
```
📍 进度更新：✅ 步骤3 完成 → 进入步骤4
```

### 设计原则

| 原则 | 说明 |
|------|------|
| 极简定位 | 仅输出当前层级位置，不展示完整列表 |
| 有向图为主 | 完整流程、分支路径由有向图展示 |
| 关键节点更新 | 仅在完成任务或重要分支时输出进度更新 |

### 与现有规范的整合

| 原有方式 | 新方式 | 原因 |
|----------|--------|------|
| TodoWrite 工具调用 | 路径追踪文本输出 | 简化输出，突出有向图 |
| 流程入口展示（四阶段概览） | **替换为阶段级有向图** | 更直观展示四阶段流转 |

## 方案2: 流程有向图设计

### 核心目的

在进入阶段/任务/步骤/流程时，展示完整的分支路径有向图，让用户一眼看到：
- 有哪些分支选择
- 每个选择后会走向哪里
- 当前处于哪个节点

### 展示时机与层级

| 层级 | 展示时机 | 展示内容 | 目的 |
|------|----------|----------|------|
| **阶段级** | 进入阶段时 | 该阶段下的所有任务 | 了解阶段任务构成 |
| **任务级** | 进入任务时 | 该任务下的所有步骤 | 了解任务步骤构成 |
| **步骤级** | 进入步骤时 | 该步骤内的判断分支 | 了解分支细节 |
| **流程级** | 执行独立流程时 | 流程内所有步骤 | 了解流程全貌 |

### 展示原则

| 原则 | 说明 |
|------|------|
| **只展开一层** | 每层级有向图仅展示下一级内容，不深入展开 |
| **进入时输出一次** | 进入该层级时输出有向图，后续不再重复输出 |
| **位置回溯即可** | 用户通过当前位置标识，向上回溯即可定位 |
| **同级不重复** | 进入步骤后，仅输出步骤级有向图，不再输出任务级图 |

### ASCII 有向图格式规范

使用 ASCII 字符绘制，确保在终端环境正常显示：

| 元素 | ASCII 表示 | 说明 |
|------|------------|------|
| 普通节点 | `[步骤1]` | 方括号包围 |
| 分支节点 | `{判断?}` | 花括号表示判断点 |
| 流程引用 | `[[流程1]]` | 双层方括号表示调用子流程 |
| 箭头 | `-->` | 单向连接 |
| 分支标注 | `-->|条件|` | 箭头上标注条件 |
| 当前节点 | `◀ 当前` | 节点后标记当前位置 |

### 绘制规范

| 规范 | 说明 |
|------|------|
| **无外框** | 不使用 `┌─┐` 方框，避免中英文混用导致对齐问题 |
| **纯文本** | 使用代码块格式，让终端自动换行 |
| **中文优先** | 所有节点和条件使用中文描述，避免技术术语 |
| **缩进对齐** | 使用空格缩进，同层级节点对齐 |
| **从上到下** | 主流程从上到下排列，分支向右延伸 |
| **无产物节点** | 有向图仅展示流程步骤，不展示输出文件（如 env.cfg） |

### 绘制模板

**模板1：线性流程**

```

```
{标题}

[步骤1] --> {判断条件?}
                │
                ├── 条件A ──→ [处理A] ──┐
                │                       │

```

**模板3：条件跳转**

```
{标题}

[步骤1] --> {需要步骤2?}
                  │
                  ├── 是 ──→ [步骤2] --> [步骤3]
                  │
                  └── 否 ──→ [步骤3]（跳过步骤2）
```

**模板4：复杂分支（多级汇聚）**

```
{标题}

[步骤1] --> {判断1?}
                │
                ├── A ──→ [处理A] ──→ {判断2?}
                │                           │
                │                           ├── X ──→ [X处理] ──┐
                │                           │                   │
                │                           └── Y ──→ [Y处理] ──┤
                │                                               │
                └── B ──→ [处理B] ──┐                          │
                                     │                          │
                                     ↓                          ↓
                              [共同后续步骤] ◀──────────────────┘
```

**符号说明**：

| 符号 | 用途 | 示例 |
|------|------|------|
| `--→` | 主流程箭头 | `[步骤1] --→ [步骤2]` |
| `──→` | 分支箭头（全角） | `├── 条件 ──→ [分支]` |
| `──┐` | 分支向下汇聚起点 | `├── A ──→ [X] ──┐` |
| `──┴─→` | 分支汇聚连接 | `└── B ──→ [Y] ──┴─→ [后续]` |
| `│` | 垂直连接线 | 连接上下节点 |
| `├` | 分支起点 | 表示分支开始 |
| `└` | 最后分支 | 表示最后一个分支 |
| `↓` | 向下箭头 | 明确流向 |
| `◀` | 当前位置标记 | `[步骤1] ◀` |
| `[[]]` | 流程引用 | `[[流程1: 问题集生成]]` |
| `{}` | 判断节点 | `{有评测集?}` |
| `⚠️` | 需确认标记 | `[步骤] ⚠️ 需用户确认` |
| `（跳过）` | 条件跳转标注 | `[步骤2]（跳过）` |

### 流程结构规范

**结构1：分支汇聚**

多个分支路径汇聚到同一节点时，使用右侧汇聚语法：

```
{条件判断}
       │
       ├── 条件A ──→ [处理A] ──┐
       │                       │
       └── 条件B ──→ [处理B] ──┴─→ [共同后续步骤]
```

**结构2：条件跳转**

满足条件时跳过某些步骤：

```
[步骤1] --> {需要步骤2?}
                  │
                  ├── 是 ──→ [步骤2] --> [步骤3]
                  │
                  └── 否 ──→ [步骤3]（跳过步骤2）
```

**结构3：条件跳转（简化表示）**

在箭头上直接标注跳转条件：

```
[步骤1] -- 满足条件 --> [步骤2]
    │
    └-- 不满足条件 --（跳过）--> [步骤3]
```

## 方案2: 流转预告机制设计

### 核心原则

| 原则 | 说明 |
|------|------|
| 预告在先 | 在用户做出选择前，说明每个选项的后续流程 |
| 回顾在后 | 在系统做出判断后，说明判断结果和后续流程 |
| 有向图优先 | 有向图已展示时，预告简化为一句话 |
| 简短聚焦 | 预告/回顾内容控制在 50 字以内 |

### 预告格式

**有向图已展示时（默认）**：
```
⚠️ 选择预告：参考上方流程图，选择后进入对应分支路径
```

**无有向图时（完整格式）**：
```
⚠️ 选择预告：
- 选择「{选项A}」→ {流程A简述}
- 选择「{选项B}」→ {流程B简述}
```

### 回顾格式

```
📋 判断结果：{value} → {后续处理简述}
```

**示例**：
```
📋 判断结果：status = partial → 询问用户确认评测集类型后调用对应流程
```

## 输出顺序规范

更新 SKILL.md 现有的"输出顺序要求"：

| 顺序 | 操作 | 说明 |
|------|------|------|
| 1 | 当前层级有向图（仅下一级内容） | 让用户看到当前层级的构成 |
| 2 | 当前位置（路径追踪） | 让用户知道"我在哪" |
| 3 | 预告（如有选择，简化为一句话） | 让用户理解"选择后的流程" |
| 4 | 执行具体步骤 | 调用脚本、读取文件等 |
| 5 | 回顾（如有判断结果，简化为一句话） | 让用户理解"判断结果和后续" |

### 输出示例

**进入评测集处理阶段时**：
```
📍 当前位置：评测集处理阶段

评测集处理阶段

[任务1: 处理评测集] --> [任务2: xxx] --> [任务3: xxx]

### 功能验证

| 验证项 | 验证方法 |
|--------|----------|

| 验证项 | 目标值 |
|--------|--------|
| 进入任务时的输出 | < 400 token（含路径追踪+有向图+预告） |
