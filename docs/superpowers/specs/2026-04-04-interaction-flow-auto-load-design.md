# 交互规范与流程感知自动加载设计

> 版本：v1.0 | 日期：2026-04-04
> 解决技能加载时交互规范和流程感知未能自动出现在上下文的问题

---

## 问题定义

### 现状

- SKILL.md 已精简至 64 行（约 1700 字）
- references/ 目录下存在完整的 `交互规范详解.md` 和 `流程感知说明.md`
- 但 SKILL.md 中缺少对这两个文档的引用链接

### 根因分析

| 问题 | 具体表现 |
|------|----------|
| **引用缺失** | SKILL.md 没有添加对 references 文档的引用 |
| **被动触发机制** | references 文档的 description 是 "Use when needing..."，Agent 需要先判断"需要"才会加载 |
| **加载时机问题** | 技能加载时只加载 SKILL.md，不会自动加载 references 目录 |

### 用户期望

技能加载时，交互规范和流程感知的核心规则应该自动出现在上下文中。

---

## 解决方案

### 方案选择

**方案A：嵌入核心规则摘要**（已采用）

- 将核心规则摘要直接嵌入 SKILL.md
- 保留详细规范在 references/ 目录
- 通过引用链接提供详细文档入口

### 设计决策

| 决策 | 说明 |
|------|------|
| 嵌入交互规范核心规则 | 禁止复合命令、必须确认的场景 |
| 嵌入流程感知核心规则 | 展示层级、状态符号 |
| 保留 references 文档 | 用于详细规范和特殊情况 |
| 简化 Red Flags 表格 | 改为"违规行为 → 后果"格式 |

---

## 实施结果

### SKILL.md 结构变化

**新增章节**：

1. **交互规范**（第32-44行）

   - 必须确认的场景列表

2. **流程感知**（第48-57行）
   - 展示层级列表

**简化章节**：

- **Red Flags**：改为简洁表格（违规行为 → 后果）
