- [ ] **Step 2: 替换流程感知章节内容**

将现有的"## 流程感知"章节替换为以下内容：

```markdown
## 流程感知

### 核心原则

| 原则 | 说明 |
|------|------|
| **渐进式披露** | 默认输出简化信息，完整信息按需展示 |
| **情境感知** | 根据用户行为动态调整输出详细程度 |
| **路径可回溯** | 展示"已走过的路径"帮助用户理解上下文 |

---

### 进度追踪 + 步骤列表

**输出时机**：进入任务时

**格式模板**：

```
📍 {阶段} > {任务} [步骤 {当前}/{总数}]

┌─────────────────────────────────┐
│ ✓ 步骤1: {名称}                 │
│ → 步骤2: {名称}                 │
│ ○ 步骤3: {名称}                 │
│ ○ 步骤4: {名称}（按需）         │
│ ○ 步骤5: {名称}                 │
└─────────────────────────────────┘
```

**状态符号**：

| 符号 | 状态 |
|------|------|
| `✓` | 已完成 |
| `→` | 当前 |
| `○` | 待执行 |
| `~` | 已跳过 |

---

### 路径记忆 + 决策记录

**输出时机**：进入有分支的步骤时

**格式模板**：

```
📍 {阶段} > {任务} > 步骤{n}: {步骤名称}

已走路径：
  {步骤n-1}: {名称} → 选择"{用户选择}"

当前判断：{判断条件}
  ├─ {选项A} → {后果A}
  ├─ {选项B} → {后果B}
  └─ {选项C} → {后果C}
```

**路径记录格式**：

| 格式 | 说明 |
|------|------|
| `步骤n: 名称 → 选择"X"` | 用户主动选择 |
| `步骤n: 名称 → 结果X` | 系统自动判断 |
| `步骤n: 名称 → 跳过` | 条件不满足跳过 |

---

### 分支树形提示

**输出时机**：遇到需要用户选择的分支点时

**格式模板**：

```
{判断/选择问题}
  ├─ {选项A} → {后果A}
  ├─ {选项B} → {后果B}
  └─ {选项C} → {后果C}
```

---

### 按需有向图

**触发方式**：

| 用户输入 | 系统响应 |
|----------|----------|
| "详细" / "流程图" / "展开" | 输出当前层级的完整有向图 |
| "简洁" / "收起" | 切换回简化模式 |

**有向图符号**：

| 符号 | 用途 |
|------|------|
| `[步骤]` | 普通节点 |
| `{判断?}` | 分支节点 |
| `[[流程]]` | 流程引用 |
| `-->` | 连接箭头 |
| `◀` | 当前位置标记 |

---

### 判断结果回顾

**输出时机**：系统完成判断后

**格式模板**：

```
📋 判断结果：{结果} → {后续处理}
```

---

### 输出顺序

| 时机 | 输出内容 |
|------|----------|
| 进入阶段 | 阶段名称 + 任务列表 |
| 进入任务 | 进度追踪 + 步骤列表 |
| 进入步骤（有分支） | 路径记忆 + 决策记录 + 分支提示 |
| 进入步骤（无分支） | 当前位置行 |
| 遇到分支判断 | 判断结果 + 后续动作 |
| 遇到用户选择 | 分支树形提示 |
| 用户请求"详细" | 完整有向图 |
```
