```markdown
### 流程有向图

**目标**：让用户一眼看到当前层级的流程结构和分支路径。

**展示原则**：

| 原则 | 说明 |
|------|------|
| 只展开一层 | 每层级有向图仅展示下一级内容，不深入展开 |
| 进入时输出一次 | 进入该层级时输出有向图，后续不再重复输出 |
| 同级不重复 | 进入步骤后，仅输出步骤级有向图，不再输出任务级图 |

**展示层级**：

| 层级 | 展示时机 | 展示内容 |
|------|----------|----------|
| 阶段级 | 进入阶段时 | 该阶段下的所有任务 |
| 任务级 | 进入任务时 | 该任务下的所有步骤 |
| 步骤级 | 进入步骤时 | 该步骤内的判断分支 |
| 流程级 | 执行独立流程时 | 流程内所有步骤 |

**格式规范**：

- 使用 ASCII 字符绘制，不使用外框
- 所有节点和条件使用中文描述
- 使用代码块格式输出

**符号说明**：

| 符号 | 用途 |
|------|------|
| `[步骤]` | 普通节点 |
| `{判断?}` | 分支节点 |
| `[[流程]]` | 流程引用 |
| `-->` | 连接箭头 |
| `◀` | 当前位置标记 |
| `⚠️` | 需确认标记 |

**绘制模板**：

```
{标题}

[步骤1] --> {判断?}
                │
                ├── 条件A ──→ [处理A] ──┐
                │                       │
                └── 条件B ──→ [处理B] ──┴─→ [后续步骤]
```
```

- [ ] **Step 3: 添加路径追踪子章节**

```markdown
### 路径追踪

**目标**：让用户知道"我在哪"。

**格式模板**：

```
📍 当前位置：{阶段} > {任务} > {步骤}
```

**输出时机**：
- 进入阶段/任务/步骤时，在有向图后输出
- 完成关键步骤后可选择性更新

**设计原则**：
- 极简定位，仅输出当前位置一行
- 完整流程由有向图展示
```

- [ ] **Step 4: 添加预告与回顾子章节**

```markdown
### 预告与回顾

**目标**：让用户理解"选择后会发生什么"。

**核心原则**：

| 原则 | 说明 |
|------|------|
| 预告在先 | 在用户做出选择前，说明后续流程 |
| 回顾在后 | 在系统做出判断后，说明结果和后续 |
| 有向图优先 | 有向图已展示时，预告简化为一句话 |

**预告格式**（有向图已展示时）：

```
⚠️ 选择预告：参考上方流程图，选择后进入对应分支路径
```

**回顾格式**：

```
📋 判断结果：{结果} → {后续处理}
```
```

- [ ] **Step 5: 更新输出顺序规范**

将现有的"输出顺序要求"表格更新为：

```markdown
### 输出顺序

| 顺序 | 操作 | 说明 |
|------|------|------|
| 1 | 当前层级有向图 | 让用户看到流程结构 |
| 2 | 当前位置（路径追踪） | 让用户知道"我在哪" |
| 3 | 预告（如有选择） | 让用户理解"选择后的流程" |
| 4 | 执行具体步骤 | 调用脚本、读取文件等 |
| 5 | 回顾（如有判断结果） | 让用户理解"判断结果和后续" |
```
